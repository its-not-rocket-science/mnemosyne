2 — RTL Arabic (ar)

Arabic starter plugin (backend/plugins/arabic.py). Dictionary mode only — no morphological analysis. Tashkeel (diacritic) stripping for canonical forms. Capabilities honestly declare morphology_depth="none" and analysis_depth="dictionary".

TC-RTL-01: Basic RTL sentence card

ذهب الطالب إلى المدرسة كل يوم.

  • [ ] Arabic text is right-aligned (text-align:start resolves to right)
  • [ ] Pills flow right → left
  • [ ] Arabic font renders (Segoe UI / SF Arabic / Noto Sans Arabic)
  • [ ] Diacritics have sufficient vertical space (line-height ≥ 1.8)
  • [ ] Inspect: lang="ar" dir="rtl" on the <p>

TC-RTL-02: Narrow viewport RTL

البيت الكبير جميل جداً.

  • [ ] Text wraps within the 20 rem box without overflow
  • [ ] Pills wrap to next line and remain right-aligned

TC-RTL-03: Mixed LTR/RTL (Arabic sentence containing a Latin proper noun)

ذهب Mohamed إلى باريس.

  • [ ] "Mohamed" and "باريس" render with correct bidirectional text (Unicode Bidi)
  • [ ] No character reversal or garbling of the Latin sequence

2b — RTL Hebrew (he)

Hebrew starter plugin (backend/plugins/hebrew.py). Dictionary mode only — no morphological analysis. Nikud (vowel-point) stripping for canonical forms. Capabilities honestly declare morphology_depth="none" and analysis_depth="dictionary". Note: inseparable prefixes (ו-, ב-, ה-, ל-…) are not split from host words.

TC-HE-01: Basic Hebrew sentence card

הילד קרא ספר מעניין בספרייה.

  • [ ] Hebrew text is right-aligned (text-align:start resolves to right)
  • [ ] Pills flow right → left
  • [ ] Hebrew font renders (Segoe UI / SF Hebrew / Noto Sans Hebrew)
  • [ ] Inspect: lang="he" dir="rtl" on the <p>

TC-HE-02: Hebrew with nikud (vowel points)

The plugin strips nikud to form canonical keys; the surface label retains the diacritics. "Base form" should appear in the lesson card when surface ≠ canonical.

בְּרֵאשִׁית בָּרָא אֱלֹהִים אֵת הַשָּׁמַיִם וְאֵת הָאָרֶץ.

  • [ ] Nikud diacritics render above/below letters without clipping
  • [ ] Line height is sufficient for diacritics (Hebrew font default)

12 — Capability-aware rendering hooks

main.js sets data-script-family, data-tokenization, and dir on sentence-card elements from plugin capabilities. These test cases use private-use language codes (x-*) to confirm that the data-attribute CSS hooks apply font and layout rules when the standard [lang^=…] selectors cannot match.

TC-HOOK-01: data-script-family="arabic" with private-use lang code

Simulates the x-rtl-test plugin output. The lang attribute is a private-use code so the [lang^="ar"] selector does NOT match. Font and line-height must come from [data-script-family="arabic"].

ذهب الطالب إلى المدرسة كل يوم.

  • [ ] Arabic font renders (data-script-family="arabic" hook applies)
  • [ ] Line height is generous (≥ 1.8) for diacritics
  • [ ] Text is right-aligned (dir="rtl" applied)
  • [ ] Pills flow right → left
  • [ ] Pill icon/badge stay in consistent LTR order; only label text is RTL
  • [ ] Inspect: lang="x-rtl-test" — NOT "ar"

TC-HOOK-02: data-script-family="cjk" with private-use lang code

Simulates the x-cjk-test plugin output. The lang attribute is a private-use code so the [lang^="zh"] selector does NOT match. word-break and letter-spacing must come from [data-script-family="cjk"].

学习这个语言非常有趣。

  • [ ] CJK system font renders (data-script-family="cjk" hook applies)
  • [ ] Characters wrap at box edge (word-break:break-all from data-script-family)
  • [ ] Letter spacing is slightly tighter (0.02em)
  • [ ] Inspect: lang="x-cjk-test" — NOT "zh"
