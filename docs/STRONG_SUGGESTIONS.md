# Strong Suggestions

## 1. Narrow the MVP
The strongest version of the MVP is:
- one language plugin
- one parse route
- one lesson modal
- one review loop
- one dashboard later

## 2. Avoid promising fluency outcomes
Keep motivational language, but avoid hard promises like "B1 in 90 days" unless validated by study data.

## 3. Treat parser confidence honestly
Morphology, idiom detection, and literary nuance should expose confidence or fallback markers.

## 4. Keep dead languages separate from the MVP
Start ancient and historical languages in annotation or dictionary mode rather than pretending robust parsing is ready.

## 5. Make accessibility measurable
Include:
- keyboard-only test path
- screen reader labels
- color-independent distinctions
- reduced motion support
- 200% zoom checks
