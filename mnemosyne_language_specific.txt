MNEMOSYNE — LANGUAGE-SPECIFIC CATALOGUE CHANGES AND FRONTEND INTEGRATION
================================================================================

Six sessions. Each builds on the previous. Run in order.

The field chain that all sessions work within:
  seed YAML
    → build_cultural_catalog.py → backend/nuance/data/cultural_references/{lang}.json
    → backend lesson API → LessonResponse.lesson_data
    → mnemosyne-detail-pane.js (reads lesson.lesson_data as `ld`)

New fields must be added at every link in this chain to reach the user.

────────────────────────────────────────────────────────────────────────────────
SCRIPT A: scripts/build_xiehouyu_csv.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
-------
Xiēhòuyǔ (歇后语) are two-part allegorical sayings where the punchline is
almost always omitted in speech. They cannot be sourced from Wiktionary (the
two-part structure is not reliably stored there) and LLM discovery treats them
as ordinary entries, missing the canonical_form_full structure.

This script uses the LLM API to generate xiēhòuyǔ entries specifically,
outputting a CSV that correctly represents the two-part structure and can be
imported via import_cultural_sources.py.

INTERFACE
---------
  python scripts/build_xiehouyu_csv.py \
    --target 200 \
    --model deepseek-chat \
    --output data/cultural_sources/zh_xiehouyu.csv \
    [--seed data/cultural_references_seed.yaml] \
    [--estimate-cost]

IMPLEMENTATION
--------------
Use the same OpenAI-compatible client pattern as extend_cultural_catalogue.py
(same .env loading, same make_client() approach, same retry wrapper).

System prompt:
  "You are a specialist in Chinese folk language and oral tradition.
  Xiēhòuyǔ (歇后语) are two-part allegorical sayings where the first part
  (setup) describes a scenario and the second part (punchline) gives the
  meaning — but the punchline is almost always implied and omitted in speech.
  Your task is to identify well-established xiēhòuyǔ in common use."

User prompt (batches of 15):
  "Generate exactly {n} well-established Chinese xiēhòuyǔ (歇后语).
  Return {\"entries\": [array of objects]}.
  
  Exclude equivalents of these already-catalogued expressions:
  {existing_sample}
  
  Each object:
    \"setup\"      -- the first part (what appears in speech), in Chinese
    \"punchline\"  -- the second implied part, in Chinese
    \"meaning\"    -- 1-2 sentences in English: what the expression means
                     and when it is used
    \"register\"   -- \"common\"|\"informal\"|\"proverbial\"
    \"confidence\" -- float 0.60-0.85 (xiēhòuyǔ origins are often disputed)
  
  Return ONLY valid JSON."

OUTPUT CSV FORMAT
-----------------
Write a CSV with EXPECTED_CSV_HEADER columns from import_cultural_sources.py.
For each entry, map fields as follows:

  language:            "zh"
  surface_pattern:     entry["setup"]  (the spoken elliptical form)
  canonical_reference: entry["setup"]
  reference_type:      "proverb_tradition"
  short_explanation:   entry["meaning"]
  source_work:         "Chinese oral tradition"
  source_author:       "Folk/oral tradition"
  source_license:      "not_required"
  rights_basis:        "common_usage_short_expression"
  register:            entry["register"]
  confidence:          entry["confidence"]
  source_dataset:      "zh_xiehouyu_generated"
  subcategory:         "xiehouyu"
  canonical_form_full: entry["setup"] + " — " + entry["punchline"]
  notes:               "Xiēhòuyǔ: setup only in surface_patterns; full form in canonical_form_full. Punchline: " + entry["punchline"]

Add a comment line at the top of the CSV:
  # Xiēhòuyǔ (歇后语) — Chinese two-part allegorical sayings.
  # surface_pattern = spoken setup only. canonical_form_full = setup — punchline.
  # Source: AI-generated; verify before promotion.

Deduplicate against the seed using the same load_existing pattern as
extend_cultural_catalogue.py (compare casefolded setup text).

Add a progress counter and cost estimate to stdout using the same token
budget approach as extend_cultural_catalogue.py:
  _XI_IN_PER_BATCH  = 500   # per batch of 15
  _XI_OUT_PER_BATCH = 900

SCRIPT B: scripts/flag_cross_language_cognates.py
────────────────────────────────────────────────────────────────────────────────

PURPOSE
-------
Japanese yojijukugo (四字熟語) and Korean sajaseong-eo (四字成語) are often
cognates of Chinese chéngyǔ with the same characters but different cultural
resonance. A user viewing 일석이조 (Korean) should be able to see that it
cognates with 一石二鸟 (Chinese). Currently there is no mechanism for this.

This script compares the Chinese, Japanese, and Korean catalogue entries and
identifies cross-language cognates, outputting a JSON file that the backend
lesson API can use to populate cross_language_cognate links.

INTERFACE
---------
  python scripts/flag_cross_language_cognates.py \
    --catalogue-dir backend/nuance/data/cultural_references \
    --output backend/nuance/data/cross_language_cognates.json \
    [--min-similarity 0.85]

IMPLEMENTATION
--------------
Load the runtime JSON for zh, ja, and ko from the catalogue directory.

Matching strategy — three passes:

  Pass 1 (exact character match): For each Japanese or Korean entry whose
  canonical_reference consists entirely of CJK unified ideographs (U+4E00–
  U+9FFF, U+3400–U+4DBF), check if any Chinese entry has a canonical_reference
  that is identical after Unicode NFC normalisation. These are definite cognates.

  Pass 2 (character-level similarity): For entries not matched in Pass 1,
  compute character-level Jaccard similarity between the canonical_reference
  character sets (each character = one token). Flag pairs with similarity ≥
  min_similarity (default 0.85) as probable cognates.

  Pass 3 (surface pattern overlap): For entries not matched in Passes 1-2,
  check if any surface_pattern from a Japanese or Korean entry exactly matches
  any surface_pattern from a Chinese entry. These are form-sharing cognates
  even if the canonical_reference strings differ.

OUTPUT FORMAT
-------------
Write a JSON file:

  {
    "generated_at": "ISO-8601 timestamp",
    "total_pairs": N,
    "cognates": [
      {
        "source_id":    "id of the ja or ko entry",
        "source_lang":  "ja" or "ko",
        "source_ref":   "canonical_reference of the ja/ko entry",
        "target_id":    "id of the zh entry",
        "target_lang":  "zh",
        "target_ref":   "canonical_reference of the zh entry",
        "match_type":   "exact"|"similar"|"surface_overlap",
        "similarity":   float (1.0 for exact, Jaccard score for similar),
        "note":         "Chinese cognate: semantic drift may apply"
      }
    ]
  }

Also write a human-readable report to stdout:
  Found {N} cognate pairs:
    exact:           {n}
    similar:         {n}
    surface_overlap: {n}

  Top 10 by similarity:
    [list]

  Pairs flagged for semantic drift review:
    (where source and target canonical_reference share characters but
     short_explanation texts have low word overlap — likely meaning shift)

The backend lesson API should load this file at startup and use it to populate
a cross_language_cognate field in lesson_data when serving phrase_family lessons
for zh/ja/ko entries that appear in the cognate file.

ACCEPTANCE CRITERIA FOR BOTH SCRIPTS
--------------------------------------
- build_xiehouyu_csv.py --target 10 produces a valid CSV that passes
  import_cultural_sources.py without validation errors.
- Each row in the CSV has canonical_form_full in "setup — punchline" format.
- flag_cross_language_cognates.py runs against the catalogue directory and
  produces cross_language_cognates.json without errors.
- For a known cognate pair (e.g. 一石二鸟/zh and 一石二鳥/ja or 일석이조/ko),
  the pair appears in the output with match_type "exact" or "similar".
- Both scripts handle missing catalogue files gracefully (skip the language
  with a warning, do not crash).

  COGNATE_FILE = Path("backend/nuance/data/cross_language_cognates.json")

  _cognate_index: dict[str, list[dict]] = {}  # entry_id → list of cognate dicts

  def _load_cognate_index() -> None:
      if not COGNATE_FILE.exists():
          return
      data = json.loads(COGNATE_FILE.read_text(encoding="utf-8"))
      for pair in data.get("cognates", []):
          sid = pair["source_id"]
          tid = pair["target_id"]
          # Index both directions so either entry can find the other
          _cognate_index.setdefault(sid, []).append({
              "id":       tid,
              "language": pair["target_lang"],
              "ref":      pair["target_ref"],
              "note":     pair.get("note", ""),
          })
          _cognate_index.setdefault(tid, []).append({
              "id":       sid,
              "language": pair["source_lang"],
              "ref":      pair["source_ref"],
              "note":     pair.get("note", ""),
          })

  _load_cognate_index()
